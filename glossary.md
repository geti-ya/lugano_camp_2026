## 기초 개념

| 용어                                           | 쉬운 설명                                                                                                               |
| -------------------------------------------- | ------------------------------------------------------------------------------------------------------------------- |
| Bitcoin                                      | 중앙 기관 없이 작동하는 디지털 화폐                                                                                                |
| 개인키 / 공개키 (Private Key / Public Key)         | 개인키는 내 돈을 쓸 수 있는 '도장'(절대 노출 금지), 공개키는 남에게 알려줘도 되는 '주소'의 재료. 개인키를 잃으면 돈에 영원히 접근 못 하고, 노출되면 돈을 뺏김                     |
| 시드 구문 / 니모닉 (Seed Phrase / Mnemonic)         | 지갑의 모든 개인키를 만들어내는 12~24개 영단어 마스터 암호. 이것만 있으면 어떤 기기에서든 지갑을 통째로 복구 가능 (단, 다중서명·타임락이 섞인 고급 지갑은 예외 → 디스크립터 필요)          |
| UTXO (미사용 거래 출력)                             | 지갑에 들어있는 개별 '동전 덩어리'. 은행 잔액처럼 하나의 숫자가 아니라 여러 개의 동전이 쌓여 있는 구조이며, 결제할 때 이 동전들을 조합해서 씀                                 |
| 사토시 (Satoshi / Sats)                         | 비트코인의 최소 단위. 1 BTC = 1억 사토시 (밀리사토시는 이를 1000분의 1로 더 쪼갠 단위)                                                           |
| 온체인 / 오프체인 (On-chain / Off-chain)            | 온체인은 비트코인 메인 장부에 직접 기록하는 거래, 오프체인은 장부 밖에서 처리한 뒤 결과만 나중에 반영하는 거래. Lightning·Ark 등 대부분의 '레이어' 기술은 오프체인 방식             |
| 노드 (Node)                                    | 비트코인 네트워크에 참여해 거래를 검증하고 장부 사본을 보관하는 컴퓨터. 누구나 직접 운영할 수 있고, 많을수록 네트워크가 더 탈중앙화됨                                        |
| 채굴 / 난이도 조정 (Mining / Difficulty Adjustment) | 채굴은 컴퓨터로 수학 퍼즐을 풀어 새 블록을 만들고 그 대가로 비트코인을 받는 과정. 난이도 조정은 채굴자가 늘어나도 블록이 항상 대략 10분마다 하나씩 생성되도록 퍼즐 난이도를 자동으로 올리고 내리는 장치 |

## 지갑 보안 & 상속

| 용어                                                | 쉬운 설명                                                                             |
| ------------------------------------------------- | --------------------------------------------------------------------------------- |
| 다중 서명 (Multisig / Multi-signature)                | 지갑 하나에 여러 개의 열쇠(개인키)를 등록해 두고, 정해진 개수 이상(예: 3개 중 2개)이 모여야만 돈을 옮길 수 있게 만든 보안 방식        |
| 타임락 (Timelock)                                    | 특정 시간이나 블록 수가 지나야만 사용할 수 있도록 코인을 묶어두는 기능. '절대적 타임락'은 특정 날짜를, '상대적 타임락'은 '마지막 이동으로부터 몇 개월'처럼 상대적 기간을 기준으로 함 |
| 디스크립터 (Descriptor)                                | 다중서명·타임락처럼 여러 열쇠와 조건이 얽힌 지갑의 '설계도' 파일. 이런 고급 지갑은 니모닉(열쇠)만으로는 복구가 안 되며, 반드시 디스크립터도 함께 백업해야 함 |
| 하드웨어 지갑 (Hardware Wallet)                         | 인터넷과 분리된 전용 기기에 개인키를 보관하고, 송금 시 진짜 소유자가 맞는지 승인(서명)할 때만 쓰는 물리적 보안 장치                    |
| 커스터디얼 / 논커스터디얼 (Custodial / Non-custodial, 비수탁형) | 커스터디얼은 거래소 등 제3자가 내 돈(개인키)을 대신 보관하는 방식, 논커스터디얼은 나 자신만 개인키를 갖고 통제하는 방식 ("Not your keys, not your coins") |

## 레이어 2 & 확장성

| 용어                        | 쉬운 설명                                                                     |
| ------------------------- | ------------------------------------------------------------------------- |
| Lightning Network         | 비트코인 거래를 빠르고 저렴하게 처리하는 레이어. 카드 결제처럼 즉시 됨                                   |
| 결제 채널 / 유동성 (Payment Channel / Liquidity) | 라이트닝에서 두 사람이 미리 돈을 걸어두고 여는 통로. '인바운드 유동성'은 내가 돈을 받기 위해 상대가 미리 채워둬야 하는 채널 잔액을 뜻함 |
| HTLC (Hashed Timelock Contract) | 라이트닝에서 "정해진 시간 안에 올바른 비밀번호(해시값)를 대야만 돈이 이동한다"는 조건을 거는 스마트 자물쇠. 중간 노드를 거쳐도 안전하게 돈을 릴레이할 수 있게 해줌 |
| 라우팅 (Routing)             | 라이트닝에서 나와 직접 연결(채널)되지 않은 상대에게, 중간 노드들을 거미줄처럼 거쳐 결제를 전달하는 과정                  |
| Liquid                     | 비트코인 위에 얹은 또 다른 레이어(사이드체인). 주로 기업 간 거래·스테이블코인 발행에 사용                       |
| 사이드체인 (Sidechain)         | 비트코인 메인 체인과 연결되어 있지만 별도 규칙으로 운영되는 보조 블록체인. Liquid가 대표 예시이며, 메인넷을 바꾸지 않고도 새 기능을 실험할 수 있음 |
| RGB                        | 비트코인 위에서 스마트컨트랙트와 토큰을 구현하는 프로토콜                                            |
| Ark                        | 비트코인의 프라이버시와 확장성을 높이기 위한 레이어                                               |
| e-cash                     | 디지털 현금. 누가 얼마를 갖고 있는지 알 수 없게 하는 기술                                         |
| CBDC (중앙은행 디지털화폐)         | 정부·중앙은행이 발행하고 통제하는 디지털 화폐. 비트코인과 반대로 개인 거래 내역을 정부가 들여다보고 통제할 수 있어 감시 도구로 악용될 수 있다는 우려가 있음 |

## 스마트 컨트랙트

| 용어                         | 쉬운 설명                                                                                         |
| -------------------------- | --------------------------------------------------------------------------------------------- |
| 스마트 컨트랙트 (Smart Contract)  | 특정 조건이 충족되면 사람의 개입 없이 자동으로 자금이 이동하도록 미리 짜둔 프로그램. 비트코인에서는 이더리움만큼 자유롭지 않은 대신 더 안전하고 예측 가능하게 설계됨 |
| Formal Verification (정형증명) | 코드가 수학적으로 올바른지 증명하는 것. 버그를 원천 차단하는 방법                                                         |
