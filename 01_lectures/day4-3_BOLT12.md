___

**2. 핵심 내용 요약 3줄**

- 기존 비트코인 결제(라이트닝 네트워크)는 항상 인터넷에 연결되어 있어야 하고, 중간 서버를 거쳐야 해서 개인정보가 노출되는 치명적인 단점이 있었다.
- 새롭게 도입된 'BOLT 12' 기술은 중간 서버 없이 사용자끼리 직접 소통하게 하여 신원을 보호하고, 오프라인 상태에서도 안전하게 결제를 주고받을 수 있게 해준다.
- 이를 통해 일반 식당의 결제 단말기(POS) 연동, 단골 고객을 위한 자동 포인트 적립(캐시백), 채팅앱을 통한 송금 등 우리가 일상에서 흔히 쓰는 편리한 결제 환경을 비트코인으로도 구현할 수 있게 되었다.

___

**3. 이 세션에서 언급된 용어 목록 + 한 줄 설명**

- **라이트닝 네트워크 (Lightning Network):** 비트코인을 실생활에서 커피를 사 먹을 수 있을 정도로 빠르고 저렴하게 결제할 수 있도록 도와주는 추가 결제 시스템.
- **BOLT 12:** 기존 비트코인 결제의 불편함(항상 온라인이어야 함, 프라이버시 부족 등)을 해결하기 위해 새롭게 만들어진 최신 결제 통신 규칙(표준)
- **오퍼 (Offer):** 한 번 쓰고 버리는 기존의 1회용 청구서와 달리, 한 번 만들어두면 웹사이트나 채팅방 등에 올려두고 여러 번 재사용할 수 있는 유연한 결제 요청서.
- **블라인디드 패스 (Blinded Path):** 결제가 전달되는 경로에 가짜 노드(거점)를 섞어 넣어, 돈을 받는 사람의 실제 신원과 위치(IP 주소 등)를 추적할 수 없게 숨겨주는 익명성 보호 기술.
- **어니언 메시지 (Onion Messages):** 카카오톡 서버 같은 중앙 서버를 거치지 않고, 네트워크 참여자들을 통해 결제 정보나 메시지를 직접 안전하게 전달하는 통신 방식.
- **LSP (Lightning Service Provider):** 일반 사용자가 항상 인터넷에 연결해두지 않아도 라이트닝 네트워크 결제를 쉽게 받을 수 있도록 연결과 관리를 대신 도와주는 서비스 제공자.

___

**4. 발표자가 특히 강조한 포인트나 인상적인 발언**

- **"결제 중간에 중앙 서버가 껴있다면, 당장 비트코인 사용을 멈추고 집에 가야 합니다."**
    - **의미:** 기존에 쓰이던 방식(LNURL 등)은 중간 서버가 누가 누구에게 얼마를 보내는지 다 알 수 있다. 발표자는 이는 비트코인의 핵심 가치인 '탈중앙화'와 '프라이버시'를 훼손하는 망가진 방식(broken)이라고 강하게 비판하며, BOLT 12를 통해 서버 없는 결제를 해야 한다고 강조했다.
- **"지금 비트코인 기반 서비스를 만들면서 BOLT 12를 모른다면, 완전히 도태된 것입니다."**
    - **의미:** BOLT 12는 단순한 기술 업데이트가 아니다. 오프라인 결제, 암호화된 고객 환불, 멤버십 포인트(캐시백) 등 상거래에 필수적인 기능을 지원하기 때문에, 앞으로의 비트코인 결제 생태계는 무조건 BOLT 12를 중심으로 돌아갈 것이라고 확신에 차서 발언했다.

**5. 흥미로웠던 영역 추가설명**

BOLT12에 대해서 잘 몰랐던 터라 뜨끔해서..ㅎㅎ 열심히 들었던 강의였다. 그래서 BOLT12에 대해 좀 더 설명한 글을 아래에 첨부한다.

단순하게 말하자면 **"비트코인을 동네 카페나 식당에서 신용카드나 삼성페이처럼 쉽고 편리하게 쓸 수 있게 만들어주는 결제 시스템"** 이라고 이해하면 된다. 발표자도 상인(Merchant)의 입장에서 이 기술이 어떻게 장사를 돕고 상거래를 원활하게 하는지 실생활 예시를 들어 설명했다.

1) BOLT12가 상거래를 도와주는 포인트

	- **단골 손님 자동 포인트 적립(캐시백)**: 복잡한 고객 데이터베이스 서버 없이도, 손님이 결제할 때마다 지갑 간 통신으로 방문 횟수를 자동 파악해 "다음 방문 시 10% 할인" 같은 혜택을 줄 수 있다.
	- **손쉬운 환불 처리**: 기존 비트코인 결제는 2달러짜리 커피가 실수로 20달러 결제되어도 이를 증명하고 환불받기가 매우 복잡해, 차라리 은행 계좌를 쓰는 게 낫다고 할 정도였다. BOLT12는 '내가 돈을 보냈다'는 확실한 암호학적 영수증(증명)이 남아 문제없이 환불을 처리할 수 있다.
	- **기존 POS 기기와의 쉬운 연동**: Square 같은 결제 단말기에 비트코인 결제 기능을 손쉽게 추가할 수 있다. 직원이 비트코인을 몰라도 현금 결제처럼 버튼 하나로 처리된다.
	- **인터넷이 끊겨도 결제 가능**: 손님의 스마트폰이 꺼져 있거나 오프라인이어도, 나중에 온라인이 되었을 때 결제를 처리할 수 있어 결제 환경의 제약이 크게 줄어든다.
	- **수수료 떼가는 중간업자 퇴출**: 기존 방식은 결제 중간에 서버가 필요했지만, BOLT12는 수수료를 떼가거나 결제 내역을 감시하는 '중간 서버'를 완전히 없앴다. 상인과 손님이 직접 돈을 주고받아 규제 리스크와 비용 부담이 줄어든다.

	결론적으로 BOLT12는 기존 비트코인의 불편한 점을 고쳐 **'상인이 매장에서 당장 쓸 수 있는 진짜 돈'** 으로 만들어주는 상거래 맞춤형 기술이다.

2) 기존 한계가 BOLT12로 어떻게 해결되었나 (기능별 설명)

	**① 암호학적 영수증 생성 및 환불 처리**  
	기존에는 결제 시 '프리이미지(preimage)'라는 단순한 증명에만 의존했는데, 이는 누구나 가로채 자신이 결제했다고 주장할 수 있어 환불 대상을 특정하기 어려웠다. BOLT12는 **'오퍼(Offer)'라는 정적이고 서명되지 않은 결제 요청서**를 도입했다. 사용자는 이 오퍼를 바탕으로 자신의 고유한 암호화 키(payer key)와 메타데이터를 담은 '인보이스 요청(invoice request)'을 직접 생성해 전달한다. 결제 완료 후 송금자는 자신이 요청서에 넣었던 키를 통해 "내가 직접 이 결제를 요청하고 돈을 보냈다"는 사실을 **암호학적으로 증명**할 수 있고, 상인도 이를 바탕으로 누구에게 돈을 돌려줘야 하는지 정확히 알 수 있어 안전한 환불이 가능해진다.

	**② 중간 서버가 필요 없어진 이유**  
	과거(LNURL 등)에는 직접 연결되지 않은 사용자 간에 인보이스를 주고받으려면 중간에 반드시 웹 서버(HTTP)를 두어야 했고, 이 서버가 누가 누구에게 얼마를 보내는지 모든 내역을 알 수 있어 프라이버시가 훼손됐다. BOLT12는 **'어니언 메시지(Onion Messages)'**라는 통신 방식을 사용해, 중앙 웹 서버 없이 라이트닝 네트워크 노드들을 통해 자금이 아닌 메타데이터·메시지만 직접 전달한다. 이를 통해 사용자와 상인이 직접 인보이스 요청을 안전하게 주고받을 수 있어 중앙 서버 자체가 필요 없어진다.

	**③ 오프라인 상태에서도 결제가 가능한 이유**  
	기존 라이트닝 네트워크는 양측 지갑이 항상 온라인이어야 한다는 제약이 있었다. BOLT12는 **'비동기 결제(Async Payments)'**로 이를 해결한다. 받을 사람의 스마트폰이 오프라인이면 **LSP(Lightning Service Provider)라는 서비스 노드가 대신 정적인 인보이스 요청을 처리**하며 결제 대기 상태를 유지하다가, 사용자가 다시 온라인이 되면 보관하던 정보를 전달해 수신을 완료시킨다. 덕분에 앱을 항상 켜두지 않아도 안전하게 돈을 받을 수 있다.

	**④ 기존 POS 기기와의 연동이 쉬워진 이유**  
	예전에는 상점이 비트코인을 받으려면 POS 기기가 직접 복잡한 라이트닝 노드를 구동하거나 무거운 API 서버를 연동해야 했다. BOLT12 환경에서는 POS 기기가 돈을 보관하거나 결제 채널을 직접 열 필요 없이, 기존 소프트웨어에 **어니언 메시지를 송수신하는 가벼운 기능만 탑재**하면 된다. 직원이 커피값 2달러를 입력하면 POS 기기는 암호화된 '결제 토큰(payment token)'을 어니언 메시지에 담아 사장님의 실제 지갑(예: Phoenix 지갑)으로 보낸다. 즉 **POS 기기는 계산과 암호화된 요청 전송만 담당하고, 실제 자금 수신은 사장님의 메인 지갑에서 독립적으로 처리**되므로 일반 결제 단말기에도 쉽게 이식할 수 있다.
